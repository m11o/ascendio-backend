# 自重トレーニング特化・提案型筋トレアプリ 仕様書 (v1.0, 2025/12/12)

## 1. コンセプト & ターゲット

- コンセプト: 「迷わない、考えない、やるだけ。」
- ターゲット: 健康志向の初心者～中級者。ジムには行かず、自宅や公園で自重トレーニングを行う層。
- コア体験: アプリを開くと「今日のメニュー」が決まっており、それをこなすだけで最適な負荷（成長）管理ができる。

## 2. メイン機能要件

| 機能 | 詳細仕様 |
| :--- | :--- |
| ホーム（提案） | 「デイリーミッション型」 UIを採用。<br>・トップ画面に「今日のクエスト（種目・回数）」を1つだけ大きく表示。<br>・「環境切替ボタン（自宅⇔公園）」を配置し、タップで即座にメニューを再計算。 |
| 記録（Input） | 「完了ボタン」＋「主観評価」 のみ。<br>・詳細なkg入力は不要。<br>・評価は3段階：😣 キツかった / 😐 普通 / 😄 余裕。<br>・この評価が次回のレベル決定アルゴリズムに直結する。 |
| ロジック（AI） | 「環境 × 回復 × プログレッション」<br>・環境: 今いる場所（自宅/公園）で可能な種目のみ抽出。<br>・回復: 昨日使った部位を避け、中2日以上の休息を確保。<br>・プログレッション: 評価が「余裕」なら、回数増ではなく**種目の難易度(Lv)**を上げる。 |
| オンボーディング | 「初回アンケート」 による初期レベル判定。<br>・3問程度の簡単な質問で、ユーザーの筋力レベル（Lv1~3）を推定する。 |

## 3. 具体的なアンケート設計案（Day 1）

案Aの詳細設計です。 ユーザーの手間を最小限にしつつ、適切な初期レベル（スタート地点）を判定するための3問を考えました。

目的: 「上半身」「下半身」それぞれの初期レベル（Lv.1～Lv.3）を振り分ける。

- Q1. 現在の運動習慣について教えてください
  - A. 全くしていない
  - B. たまに歩いたり、軽い運動をする
  - C. 定期的に運動している（週2回以上）
  - → 判定: Cなら全体の基準値を少し上げる
- Q2. 「腕立て伏せ」について
  - A. 膝をつかずに10回以上できる
  - B. 膝をつけば10回できる
  - C. 10回は厳しい / 自信がない
  - → 判定: A=通常プッシュアップ(Lv.3) / B=膝つき(Lv.2) / C=壁プッシュアップ(Lv.1) へ振り分け
- Q3. 「スクワット（しゃがみ込み）」について
  - A. 深くしゃがんで、何回でもできる自信がある
  - B. 10回くらいなら問題ない
  - C. 膝や腰に不安がある
  - → 判定: A=ランジなど高負荷(Lv.3) / B=通常スクワット(Lv.2) / C=椅子スクワット(Lv.1) へ振り分け
